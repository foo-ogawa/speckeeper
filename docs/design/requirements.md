---

## CR-001: Minimize External Dependencies

**Type**: constraint | **Priority**: should | **Category**: technical

Limit external dependencies to stable libraries like Zod, Commander, Chalk

### Acceptance Criteria

- **CR-001-01**: Keep dependency package count under 20 [inspection]
- **CR-001-02**: Use only packages with stable major versions [review]

---

## CR-002: Zero-config Startup

**Type**: constraint | **Priority**: should | **Category**: usability

Basic functionality works without config file

### Acceptance Criteria

- **CR-002-01**: build/lint/drift works without speckeeper.config.ts [test]
- **CR-002-02**: Default settings cover typical use cases [review]

---

## CR-003: No Implementation Code Generation

**Type**: constraint | **Priority**: must | **Category**: scope

speckeeper does not generate implementation code. API contracts and DB connection code are handled by external tools

### Acceptance Criteria

- **CR-003-01**: speckeeper does not generate or modify code under src/ [review]
- **CR-003-02**: API contracts (TypeScript types, routes, clients) generated by external tools from OpenAPI [review]
- **CR-003-03**: DB connections (Entity types, repositories) generated by ORM/DDL tools from DDL/schema [review]
- **CR-003-04**: speckeeper only handles requirement definition and external SSOT consistency checks [review]

---

## CR-004: Respect External SSOT

**Type**: constraint | **Priority**: must | **Category**: scope

Treat API specs, DB definitions, IaC definitions as external SSOT, speckeeper only references and verifies

### Acceptance Criteria

- **CR-004-01**: Do not directly generate or modify OpenAPI files [review]
- **CR-004-02**: Do not directly generate or modify DDL/Prisma schemas [review]
- **CR-004-03**: Do not directly generate or modify CloudFormation/Terraform [review]
- **CR-004-04**: Principle is to fix speckeeper side when consistency check fails [review]
